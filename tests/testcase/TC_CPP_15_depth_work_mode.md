## 关联功能点

本模块覆盖 [sdk_feature_matrix.md](../sdk_feature_matrix.md) 中 **模块 15：深度工作模式** 的全部功能点：

| 编号 | 功能点 | C API | C++ API |
|------|--------|-------|---------|
| 15-010 | 获取当前模式 | `ob_device_get_current_depth_work_mode()` | `getCurrentDepthWorkMode()` |
| 15-020 | 获取当前模式名称 | `ob_device_get_current_depth_work_mode_name()` | `getCurrentDepthWorkModeName()` |
| 15-030 | 枚举模式列表 | `ob_device_get_depth_work_mode_list()` | `getDepthWorkModeList()` |
| 15-040 | 按名称切换模式 | `ob_device_switch_depth_work_mode_by_name()` | `switchDepthWorkModeByName()` |
| 15-050 | 按 struct 切换模式 | `ob_device_switch_depth_work_mode()` | `switchDepthWorkMode()` |

### 基本信息

| 字段 | 值 |
|------|-----|
| 编号 | TC_CPP_15_01 |
| 名称 | 获取当前模式与名称 |
| 目的 | 验证可获取当前深度工作模式和名称 |
| 覆盖功能点 | 15-010, 15-020 |
| 前置条件 | 至少一台设备通过 USB 连接 |
| 硬件依赖 | 是 |
| 优先级 | P0 |

### 测试步骤

1. 获取设备
2. 调用 `getCurrentDepthWorkMode()`
3. 调用 `getCurrentDepthWorkModeName()`

### 检查项

- ✅ 返回有效的 OBDepthWorkMode 结构体
- ✅ 模式名称非空字符串
- ✅ 名称描述清晰（如 "256x192 @ 30fps"）

### 预期结果

当前深度工作模式和名称正确获取。
